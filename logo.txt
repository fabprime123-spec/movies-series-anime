This prompt sets up Logo.dev in your Android (Kotlin) codebase to load and display company logos by domain (e.g., `stripe.com`) using your free tier publishable key (`pk_cxnUR6VpRn-IQV46IhPFpQ`). 

Because you are on the free tier, commercial applications must include a visible attribution link to `https://logo.dev`. Personal projects are exempt.

### 1. Modern Image Loading (Coil)
If your project uses [Coil](https://coil-kt.github.io/coil/) (recommended for modern Android apps), load the logo directly into an `ImageView` using the [Logo.dev Image CDN](https://www.logo.dev/docs/migrations/clearbit#migrate-your-website-or-app):

```kotlin
import coil.load

fun loadCompanyLogo(domain: String, imageView: ImageView) {
    // Uses your Logo.dev publishable key safe for client-side use
    val url = "https://img.logo.dev/$domain?token=pk_cxnUR6VpRn-IQV46IhPFpQ"
    
    imageView.load(url) {
        placeholder(R.drawable.logo_placeholder)
        error(R.drawable.logo_error)
    }
}
```

### 2. Traditional Image Loading (Picasso)
If you prefer [Picasso](https://square.github.io/picasso/), use this implementation:

```kotlin
import android.widget.ImageView
import com.squareup.picasso.Picasso

fun loadCompanyLogo(domain: String, imageView: ImageView) {
    // Uses your Logo.dev publishable key safe for client-side use
    val url = "https://img.logo.dev/$domain?token=pk_cxnUR6VpRn-IQV46IhPFpQ"
    
    Picasso.get()
        .load(url)
        .placeholder(R.drawable.logo_placeholder)
        .error(R.drawable.logo_error)
        .into(imageView)
}
```

### 3. Required Attribution (Free Tier Only)
If this is a commercial app, display a visible, followable attribution link in your UI:
* **Web/HTML output:** `<a href="https://logo.dev" title="Logo API">Logos provided by Logo.dev</a>`
* **App UI:** A clear text label indicating "Logos provided by Logo.dev" linking to `https://logo.dev`.
